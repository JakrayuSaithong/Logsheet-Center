# ระบบ Log Sheet ใบนำของออกนอกบริษัท

ระบบนี้ถูกพัฒนาขึ้นเพื่อใช้สำหรับจัดการใบนำของออกนอกบริษัท (Gate Pass) ของพนักงาน โดยรองรับการสร้าง แก้ไข ตรวจสอบสถานะการนำกลับ และรวมถึงการออกรายงานเป็นรูปแบบไฟล์ PDF

## การออกแบบระบบ (System Design)
ระบบถูกออกแบบโดยใช้สถาปัตยกรรมแบบ **Vanilla PHP + Custom Router** เพื่อให้ทำงานได้อย่างรวดเร็วและใช้ทรัพยากรน้อย ผสานรวมกับการจัดการ Database สองระบบคือ **SQL Server** สำหรับเก็บข้อมูลหลัก และ **MySQL** สำหรับเชื่อมต่อข้อมูลพนักงาน

หน้าต่างผู้ใช้งาน (UI) ถูกพัฒนาขึ้นมาเพื่อเน้นประสบการณ์การใช้งานที่ลื่นไหล (UX) โดยมีการใช้งาน Modal เพื่อความรวดเร็วในการกรอกข้อมูล และมี DataTable ในการจัดการแสดงผลข้อมูลจำนวนมาก

### ระบบความปลอดภัย
ที่โดดเด่นคือการออกแบบระบบ **File Upload Security** แบบหลายชั้น เพื่อป้องกันช่องโหว่ประเภท File Upload Vulnerability (เช่น Webshell, malicious PHP, Double Extension) ซึ่งจะตรวจจับลึกถึงระดับ MIME Type จริงโดยไม่สนใจนามสกุลไฟล์ และตรวจสอบเนื้อหา Payload ภายในไฟล์และ ZIP เพื่อป้องกันโค้ดอันตราย

## อธิบายการทำงานและโครงสร้างไฟล์ (Files functionality)

การทำงานของระบบถูกจัดเรียงให้อยู่ในรูปแบบ Modular ซึ่งแบ่งการทำงานออกเป็นไฟล์ๆ ดังนี้:

### 1. ระบบจัดการเส้นทางและโครงสร้างหลัก (Core & Routing)
* `index.php`: เป็นจุดศูนย์กลางของแอปพลิเคชัน (Entry Point) ดำเนินการตรวจสอบ Session, ตรวจสอบสิทธิ์ผู้ใช้งานจาก External API, และระบุเส้นทาง URL ต่างๆ
* `route.php`: คลาสสำหรับจัดการ Custom Routing (กำหนด URL path เช่น `/insert_logsheet` หรือ `/report`)
* `config/base.php`: ไฟล์ศูนย์รวมสำหรับการเรียกใช้ระบบฐานข้อมูลและฟังก์ชันพื้นฐานที่จำเป็น

### 2. การเชื่อมต่อฐานข้อมูลและความปลอดภัย (Database & Security)
* `config/connection.php`: คลาสเชื่อมต่อฐานข้อมูล รองรับทั้ง `AsefaLogSheet` (SQL Server) และ `rypsoftcom_erp2` (MySQL)
* `config/security.php`: ระบบตรวจจับไฟล์อันตราย (Webshell Detection) แบบหลายชั้น ป้องกันการอัปโหลดไฟล์ที่มีแฝงโค้ดรูปแบบ .php หรือไฟล์ที่ปลอมแปลง MIME types
* `config/function.php`: เก็บฟังก์ชันส่วนกลางของระบบและการทำงานของ Logic หลักซ้ำๆ

### 3. ส่วนแสดงผลต่อผู้ใช้งานและจัดการข้อมูล (Views & Controllers)
* `home.php` และ `test.php`: หน้าหลักของระบบ แสดงรายการเอกสารใบนำของออกในรูปแบบ DataTable และมี Modal สำหรับเรียกใช้งานฟอร์มเพิ่มข้อมูลเพิ่ม/ค้นหา
* `InsertForm.php` และ `EditForm.php`: หน้าสำหรับสร้างและและแก้ไขรายละเอียดของใบนำของออกแบบเต็มรูปแบบ
* `doc_Refer_list.php`, `item_list.php`, `type_doc.php`, `sheet_status_list.php`: หน้าจัดการ Master Data (ข้อมูลอ้างอิงของระบบ) เช่น รายการนำออก, ประเภทเอกสาร, สถานะ, ฯลฯ
* `permission.php`: ส่วนที่ใช้กำหนดสิทธิ์การเข้าถึงข้อมูลของพนักงานและอนุมัติการทำงาน
* `report.php`: หน้าแสดงผลรายงานแบบ Dashboard

### 4. ระบบออกรายงานและการพิมพ์ (PDF Generation)
* `Print_PDF.php` และ `Print_PDF_New.php`: ชุดไฟล์สำหรับการดึงข้อมูลจากฐานข้อมูลมาวาดและประมวลผลเป็นไฟล์ PDF เพื่อให้ผู้ใช้งานสามารถสั่งพิมพ์ (Print) เพื่อนำไปใช้งานได้จริง

### 5. ทรัพยากรและองค์ประกอบอื่นๆ (Assets & Layouts)
* `layout/`: ประกอบด้วยองค์ประกอบย่อยของ UI เช่น `navbar.php`, `meta.php`, `css.php`, `js.php` เป็นต้น เพื่อนำไปรวมร่างประกอบเป็นหน้าเว็บไซต์
* `assets/`: โฟลเดอร์ที่รวบรวมไฟล์สไตล์ชีท (CSS), สคริปต์ (JS), Plugins ตัวช่วยต่างๆ รวมถึง Fonts และรูปภาพพื้นฐาน
* `icon/`: แหล่งเก็บทรัพยากรรูปภาพขนาดเล็ก เช่น ไอคอนสถานะ
